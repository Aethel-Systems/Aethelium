# Aethelium 块量（`pbit`）设计规范

## 1. 核心概念与声明语法

`pbit` 作为基本类型/关键字，用来标识**原始内存位/地址的操作实体**。根据声明后缀和修饰符的不同，它分为**单体指针**与**块量**两种形态。

### 1.1 单体指针（Singular Pointer）
表示一个精确的、单一的内存地址。
```ae
// 声明 a 存储单体地址 0x1000
pbit a = 0x1000
```
* **行为**：此时 `a` 仅代表单个地址起点，通常只能以单体形式或者转译为普通 `ptr` 访问。

### 1.2 块量（Block Quantity）
表示一段**拥有明确起点和终点的连续内存区间**。为了突出其“块”的属性并与单体指针区分，提供以下两种语法设计：

#### 形式 A：后缀快速声明（`"` 后缀）
在变量标识符后加上 `"` 代表这是一个块量，使用 `'` 隔开起止地址（空格可有可无）：
```ae
// 声明块量 b，代表 [0x2, 0x12] 之间的连续内存
pbit b" = 0x2 ' 0x12
```

#### 形式 B：区间结构声明（冒号 `:[]`）
使用冒号配合中括号来突出块量的区间属性，使得多变量或动态赋值时更加清晰：
```ae
// 声明块量 b，显式锁定 0x2 到 0x12 区间
pbit b: [0x2 ' 0x12]

// 也支持动态变量作为边界
pbit dynamic/block: [start/addr ' end/addr]
```

## 2. 块量（`pbit`）的核心操作与功能设计

为了发挥块量真正的优势，我们在设计中为其注入以下原生语法操作：

### 2.1 隐式元数据读取（No-Overhead Reflection）
块量在编译期/运行期自带两个轻量元数据属性，无需开发者手动维护大小：
* `b\start`：获取块的起始地址（如 `0x2`）。
* `b\end`：获取块的结束地址（如 `0x12`）。
* `b\size`：自动计算块的区间大小（即 `end - start + 1`）。

### 2.2 自动边界安全校验（Bounds Checking）
当使用下标访问块量内部内容时，编译器/运行时自动插入边界检查，防止越界：
```ae
pbit b: [0x1000 ' 0x1010] // 大小为 17 字节

// 安全：在 [0x1000 ' 0x1010] 范围内
b[0] = 0xAA 

// 报错/拒绝执行：下标 20 已经超出了该块量的安全终点 0x1010
b[20] = 0xBB 
```

### 2.3 原生区间裁剪与滑移（Sub-Slicing）
块量内部可以使用冒号 `:[]` 配合 `'` 进行二次区间裁剪，生成新的子块量：
```ae
pbit parent: [0x1000 ' 0x2000]

// 裁剪出 parent 块中偏移量从 0x10 到 0x50 的子块
pbit child: parent:[0x10 ' 0x50]
```

## 3. 针对 Wiper（擦除器）场景的特色增强功能

块量概念非常适合高安全性的数据擦除和内存覆写。我们可以为其设计以下高级特性：

### 3.1 一键物理擦除（Block Wiping）
无需写循环，块量本身支持一键填充（Wipe），直接映射为高效的 CPU 填充指令（如 `rep stos` 或 ARM 向量填充）：
```ae
pbit b: [0x1000 ' 0x2000]

// 一键将整块内存擦除为 0
b:[] = 0

// 用特定安全字符 0x55 覆写整块内存
b:[] = 0x55
```

### 3.2 块间安全拷贝（Block Copy）
直接通过块量赋值进行内存拷贝，编译器会自动校验目标块和源块的 `size`，防止缓冲区溢出，并自动处理内存重叠（Overlap）：
```ae
pbit src: [0x1000 ' 0x1050]
pbit dest: [0x2000 ' 0x2050]

// 安全拷贝：编译器校验双方 size 一致后执行高效拷贝
dest:[] = src:[]
```

### 3.3 常时对比（Constant-time Compare）
用于比对两块内存是否一致（例如文件哈希、签名或校验码验证），避免由于分支预测导致的侧信道攻击：
```ae
pbit key1: [0x1000 ' 0x1020]
pbit key2: [0x2000 ' 0x2020]

// 返回 Bool 值
if key1:[] == key2:[] {
    print("Match!")
}
```
