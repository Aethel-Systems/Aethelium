# AethelOS 硬件层工业手册（机器码逐条对照版）

版本：2026-03-06  
文档范围：**仅覆盖当前仓库中已经实现并可编译的硬件层能力**（不包含未落地设计项）。  
主要依据：
- `examples/HardwareLayerComprehensive.ae`
- `toolsC/compiler/src/frontend/hardware_layer.c`
- `toolsC/compiler/src/frontend/hardware_gates.c`
- `toolsC/compiler/src/middleend/codegen/aec_codegen.c`

---

## 1. 文档目标

本手册用于三个场景：

1. 内核/引导链开发时，快速确认某条硬件层语句会下发什么机器码。  
2. 排查“能编译但行为不对”时，对照实际字节流核对语义。  
3. 做硬件层代码评审时，判断写法是否走到了已实现、已验证路径。  

---

## 2. 快速验证流程（建议固定执行）

### 2.1 编译综合样例

```bash
./build/output/aethelc examples/HardwareLayerComprehensive.ae -o /tmp/hwTest.bin --emit bin --bin-flat
```

### 2.2 反汇编

```bash
ndisasm -b 64 /tmp/hwTest.bin
```

### 2.3 关键字检查

```bash
ndisasm -b 64 /tmp/hwTest.bin | rg "syscall|in al|pause|cli|sti|hlt|mfence|mov rax,cr3|mov cr3,rax|rdmsr|wrmsr|lgdt|cpuid|retf"
```

## 3. 硬件层语法总览（当前可用）

1. `hardware func ... { ... }`  
2. `hardware { ... }`  
3. `@gate(type: \interrupt|\syscall|\efi|\naked|\exception)`  
4. `reg<"...", T>` 物理寄存器绑定  
5. `port<T>` 端口抽象  
6. `@volatile view<T>` MMIO视图  
7. `CPU/Current\Control\CRx` 控制寄存器读写  
8. `CPU/Flags\...` 标志位读写  
9. `CPU/MSR\...` MSR路径访问  
10. `hardware\isa\op()` ISA直通  
11. `morph { ... goto ... }` 形态置换  
12. `hardware\state\snapshot(into: ...)` 上下文快照  
13. `vector/read` 与 `vector/write` 向量访存

## 4. ISA 直通与机器码对照（前端映射）

以下为 `hw_generate_isa_opcode()` 当前已实现映射。

| 硬件层调用 | 机器码 | 说明 |
|---|---|---|
| `hardware\isa\syscall()` | `0F 05` | 系统调用入口 |
| `hardware\isa\cpuid()` | `0F A2` | CPU 特性查询 |
| `hardware\isa\sysret()` / `sysretq()` | `48 0F 07` | 64位系统调用返回 |
| `hardware\isa\iret()` / `iretq()` | `48 CF` | 64位中断返回 |
| `hardware\isa\iretd()` | `CF` | 32位返回形态 |
| `hardware\isa\int3()` | `CC` | 调试中断 |
| `hardware\isa\cli()` | `FA` | 关中断 |
| `hardware\isa\sti()` | `FB` | 开中断 |
| `hardware\isa\pause()` | `F3 90` | 自旋等待友好暂停 |
| `hardware\isa\hlt()` | `F4` | 停机等待中断 |
| `hardware\isa\mfence()` | `0F AE F0` | 全栅栏 |
| `hardware\isa\lfence()` | `0F AE E8` | 读栅栏 |
| `hardware\isa\sfence()` | `0F AE F8` | 写栅栏 |
| `hardware\isa\monitor()` | `0F 01 C8` | MONITOR |
| `hardware\isa\mwait()` | `0F 01 C9` | MWAIT |
| `hardware\isa\wbinvd()` | `0F 09` | 回写并失效cache |
| `hardware\isa\invd()` | `0F 08` | 失效cache |
| `hardware\isa\nop()` | `90` | 空操作 |

说明：
- `lgdt/lidt/goto` 在 codegen 中有专门处理，不完全走上表通路，见第 9 章。

## 5. 控制寄存器/标志寄存器机器码对照

## 5.1 控制寄存器（CR0/2/3/4/8）

| 硬件层语义 | 机器码模板 | 备注 |
|---|---|---|
| 读 `CRx` 到 `RAX` | `48 0F 20 /r` | 例如 `CR3 -> RAX` 常见为 `0F 20 D8`（含/不含REX由路径决定） |
| 写 `RAX` 到 `CRx` | `48 0F 22 /r` | 例如 `MOV CR3,RAX` 为 `48 0F 22 D8` |

实测样例（`HardwareLayerComprehensive.ae`）：
- `480F20D0` -> `mov rax,cr2`
- `480F22D8` -> `mov cr3,rax`
- `0F20D8` -> `mov rax,cr3`（特定路径可不带 `48`）

## 5.2 RFLAGS

| 硬件层语义 | 机器码 | 备注 |
|---|---|---|
| 读全部标志 `CPU/Flags\Value` | `9C 58` | `pushfq; pop rax` |
| 写全部标志 `CPU/Flags\Value = x` | `50 9D` | `push rax; popfq` |
| `CPU/Flags\Interrupt/Enable=false` | `FA` | `cli` |
| `CPU/Flags\Interrupt/Enable=true` | `FB` | `sti` |

## 6. 端口 I/O 机器码对照

`AST_HW_PORT_IO` 由 codegen 直接发码，关键路径如下。

## 6.1 读端口

| 位宽 | 机器码路径 | 说明 |
|---|---|---|
| 8-bit | `EC imm8` 或 `E4 imm8`（不同路径） | `in al, port` |
| 16-bit | `66 ED imm8` | `in ax, ...` |
| 32-bit | `ED imm8` | `in eax, ...` |
| 64-bit | `48 ED imm8` | `in rax, ...` |

> 注：当前项目里“端口常量 + `\read()`”路径与“AST_HW_PORT_IO”路径并存，最终字节可能不同但语义一致。

## 6.2 写端口

| 位宽 | 机器码路径 | 说明 |
|---|---|---|
| 8-bit | `EE imm8` | `out port, al` |
| 16-bit | `66 EF imm8` | `out port, ax` |
| 32-bit | `EF imm8` | `out port, eax` |
| 64-bit | `48 EF imm8` | `out port, rax` |

实测样例：
- `E4 64` -> `in al,0x64`
- `E4 60` -> `in al,0x60`

---

## 7. MSR 机器码对照

| 硬件层语义 | 机器码 | 备注 |
|---|---|---|
| `rdmsr` | `0F 32` | `ECX` 为 MSR 索引，结果在 `EDX:EAX` |
| `wrmsr` | `0F 30` | `ECX` 索引，写入 `EDX:EAX` |

专项路径（`CPU/MSR\...`）：
- `LSTAR`
- `STAR`
- `EFER/Syscall/Enable`

这些路径在 codegen 中会自动补充 `ECX` 索引和位操作序列，不要求手写裸 `rdmsr/wrmsr`。

## 8. 向量化（YMM）机器码对照

当前综合样例使用：
- `vector/read<UInt8,32>`
- `vector/write<UInt8,32>`
- `reg<"ymm0"... "ymm3"...>`

发码主模板（当前实现）：

| 操作 | 字节前缀 | 说明 |
|---|---|---|
| 读向量到 YMM | `C5 FD 6F /r` | 256-bit load |
| 写 YMM 到内存 | `C5 FD 7F /r` | 256-bit store |

实测样例：
- `C5FD6F00`
- `C5FD6F08`
- `C5FD6F10`
- `C5FD6F18`
- `C5FD7F00`

## 9. 特殊控制流机器码对照（非普通函数调用）

## 9.1 `goto target`

1. 普通路径：`FF E0`（`jmp rax`）  
2. 若之前设置了 `reg<"cs"> = selector`，则走 far-transfer 路径：
   - `B9 xx xx xx xx`（加载 selector）
   - `51`（push rcx）
   - `50`（push rax，目标RIP）
   - `CB`（`retf`）

实测可见：
- `CB`（`retf`）

## 9.2 `hardware\isa\lgdt(ptr)`

codegen 专项处理为：
- `0F 01 10`（`lgdt [rax]` 形态）

实测可见：
- `0F0110` -> `lgdt qword [rax]`

## 10. `hardware\state\snapshot` 机器码语义

该 API 用于线程切换保存上下文，当前实现覆盖：

1. GPR 集写入 `ThreadContext` 字段偏移。  
2. `rsp` 保存。  
3. `rip` 保存（`call` + `pop` 获取当前位置）。  
4. `cr3` 保存到 `page/table` 字段。  

这意味着 `thread/switch` 场景中“保存旧线程上下文”是实装路径，不是空操作。

---

## 11. `@gate` 类型与返回指令关系

| gate 类型 | 当前返回语义 |
|---|---|
| `\interrupt` | `iretq` 路径 |
| `\exception` | `iretq` 路径 |
| `\syscall` | `sysretq` 路径（无显式 return 时） |
| `\efi` | 标准函数尾声（`leave; ret`） |
| `\naked` | 不自动生成标准序言/尾声 |

## 12. 从源码到机器码：逐段对照示例

以下示例取自 `examples/HardwareLayerComprehensive.ae` 的典型片段。

## 12.1 系统调用准备

源码关键语义：
- 绑定 `rax/rdi/rsi/rdx`

实测关键字节：
- `0F 05` -> `syscall`

## 12.2 PS/2 轮询

源码关键语义：
- `status = PS2/STATUS/PORT\read()`
- 条件判断后 `hardware\isa\pause()`

实测关键字节：
- `E4 64` -> `in al,0x64`
- `F3 90` -> `pause`

## 12.3 缺页地址读取

源码关键语义：
- `fault/addr = CPU/Current\Control\CR2`

实测关键字节：
- `48 0F 20 D0` -> `mov rax,cr2`

## 12.4 中断临界区

源码关键语义：
- 保存 `CPU/Flags\Value`
- `CPU/Flags\Interrupt/Enable = false`
- `hardware\isa\mfence()`

- `9C 58` -> `pushf; pop rax`
- `FA` -> `cli`
- `0F AE F0` -> `mfence`

## 12.5 分页切换

源码关键语义：

## 12.6 AVX memcpy

源码关键语义：
- 连续 `vector/read` + `vector/write`

实测关键字节：
- `C5 FD 6F xx`（load）

## 13. 结构体字段访问与偏移规则（当前实现要点）

1. codegen 会收集结构体布局并计算字段偏移。  
2. `ptr<T>`/`view<T>` 字段读写按偏移直接访问。  
3. 支持 `field[i]` 和嵌套索引形式。  
4. 兼容路径中存在对“去首字母写法”的回退匹配（历史源码兼容），但工程上推荐始终使用精确字段名。  

---

## 14. 推荐工程写法（工业实践）

1. 所有架构副作用语句放入 `hardware {}`。  
2. 中断/异常入口强制使用 `@gate(type: \interrupt|\exception)`。  
3. `CRx/Flags/MSR` 操作封装为独立 `hardware func`。  
4. 上下文切换统一模式：`snapshot` -> `morph` -> `goto`。  
5. SIMD 复制统一模式：向量块 + 尾字节循环。

## 15. 常见故障与定位方法

## 15.1 `Failed to emit function machine code: ...`

定位顺序：
1. 先把目标函数单独抽到最小文件。  
2. 保留 `hardware` 主路径，去掉无关逻辑。  
3. 核对寄存器名是否在支持集。  
4. 核对结构体字段名与定义一致。  
5. 反汇编确认目标指令是否实际存在。  

## 15.2 机器码“编过但不对”

定位顺序：
1. `ndisasm -b 64` 看实际字节。  
2. 本手册对照表核对指令编码。  
3. 若是门函数，确认 gate 类型是否正确。  
4. 若是 `goto`/段寄存器切换，确认是否触发 `retf` 路径。

## 18. 新增 CPU 模式一行跳转（2026-03-08）

## 18.1 `hardware\isa\modejump32(entry[, loadBase])`

适用范围：
- 仅在 16 位机器模式（`--machine-bits 16`）可用。
- 省略 `loadBase` 时默认 `0x7C00`。

自动发码序列：
1. `cli`  
2. 自动构建并加载内置 `GDTR/GDT`（`lgdt [disp16]`）  
3. 打开 `CR0.PE`  
4. 远跳转到内部 32 位入口（`66 EA ptr16:32`）  
5. 重载 `DS/ES/FS/GS/SS` 到 `0x10`  
6. 设置 `ESP`  
7. 跳到 `entry`（32 位线性地址）  

关键字节：
- `0F 01 16`（`lgdt [disp16]`）  
- `66 0F 20 C0`、`66 0F 22 C0`（`mov eax,cr0` / `mov cr0,eax`）  
- `66 EA ... 08 00`（远跳转到 32 位代码段）  

## 18.2 `hardware\isa\modejump64(entry[, loadBase])`

适用范围：
- 仅在 32 位机器模式（`--machine-bits 32`）可用。
- 省略 `loadBase` 时默认 `0x8000`。

自动发码序列：
1. `cli`  
2. 自动构建并加载内置 `GDTR/GDT`（`lgdt [disp32]`）  
3. 打开 `CR4.PAE`  
4. 通过 `rdmsr/wrmsr`（`MSR 0xC0000080`）打开 `EFER.LME`  
5. 自动构建页表并写入 `CR3`  
6. 打开 `CR0.PG|PE`  
7. 远跳转到内部 64 位入口  
8. `RAX=entry` 后 `jmp rax`  

关键字节：
- `0F 01 15`（`lgdt [disp32]`）  
- `0F 20 E0`、`0F 22 E0`（`mov eax,cr4` / `mov cr4,eax`）  
- `0F 32`、`0F 30`（`rdmsr` / `wrmsr`）  
- `0F 22 D8`（`mov cr3,eax`）  
- `EA ... 08 00`（远跳转）  
- `48 B8 <imm64> FF E0`（`mov rax,imm64; jmp rax`）  

## 18.3 BIOS 用法

- Stage1 实模式 -> 保护模式：  
  - `hardware { hardware\isa\modejump32(0x8000) }`
- Stage2 保护模式 -> 长模式：  
  - `hardware { hardware\isa\modejump64(0x1000000) }`
